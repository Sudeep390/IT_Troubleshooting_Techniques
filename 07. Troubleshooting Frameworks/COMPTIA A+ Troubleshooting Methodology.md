
# 🛠 CompTIA Troubleshooting Methodology

## Overview

A structured approach used by IT professionals to diagnose and resolve technical issues.

---

# 7-Step Troubleshooting Process

## 1. Identify the Problem

Collect information:

- Ask the user questions
- Identify symptoms
- Check error messages
- Determine impact

---

## 2. Establish a Theory of Probable Cause

Consider possible causes:

Example:

No internet connection:

- Cable disconnected
- DHCP failure
- DNS issue
- Router problem

---

## 3. Test the Theory

Use tools:
ping
ipconfig
Event Viewer
Task Manager


Confirm the root cause.

---

## 4. Establish a Plan of Action

Decide:

- Best solution
- Required permissions
- Possible risks

---

## 5. Implement the Solution

Apply the fix.

Examples:

- Update driver
- Reset password
- Restart service

---

## 6. Verify Functionality

Confirm:

- Issue resolved
- System works correctly
- User can continue working

---

## 7. Document Findings

Record:

- Problem
- Cause
- Solution
- Prevention steps

---

# Example

Issue:

User cannot access email.

Cause:

Corrupted Outlook profile.

Solution:

Created a new Outlook profile.

Result:

Email restored.
